To handle PDF uploads with security, you apply the same **RLS (Row Level Security)** logic to Supabase **Storage** as you did for your database tables.

In Supabase, storage files are tracked in a table called `storage.objects`. You can write policies that look at the file path to determine if a user has permission to upload or view a file.

### 1. Storage RLS Policies (SQL)
Run these in your Supabase SQL Editor. This setup assumes your files are organized by folder name (e.g., `faculty-docs/user_id/resume.pdf`).

```sql
-- 1. Create a private bucket
-- (You can also do this in the Dashboard UI)
insert into storage.buckets (id, name, public) 
values ('faculty-docs', 'faculty-docs', false);

-- 2. Allow users to upload files ONLY to their own folder
create policy "Users can upload to their own folder"
on storage.objects for insert
with check (
  bucket_id = 'faculty-docs' AND 
  (storage.foldername(name))[1] = auth.uid()::text
);

-- 3. Allow users to view ONLY their own files
create policy "Users can view their own files"
on storage.objects for select
using (
  bucket_id = 'faculty-docs' AND 
  (storage.foldername(name))[1] = auth.uid()::text
);

-- 4. Allow admins to view EVERYTHING
create policy "Admins can view all files"
on storage.objects for select
using (
  bucket_id = 'faculty-docs' AND 
  (auth.jwt() -> 'app_metadata' ->> 'role') = 'admin'
);
```

### 2. FastAPI Upload Implementation
We need to use the user's **JWT token** when calling the storage upload. This ensures that if a user tries to upload a file to someone else's folder, Supabase will reject it at the database level.

```python
from fastapi import FastAPI, Depends, UploadFile, File, HTTPException, Header
from supabase import create_client, Client

SUPABASE_URL = "https://your-project.id.supabase.co"
SUPABASE_ANON_KEY = "your-anon-key"

app = FastAPI()

# Helper to get user info from the token
async def get_user_id(authorization: str = Header(...)):
    token = authorization.split(" ")[1]
    client = create_client(SUPABASE_URL, SUPABASE_ANON_KEY)
    user = client.auth.get_user(token)
    return user.user.id, token

@app.post("/upload-pdf")
async def upload_faculty_pdf(
    file: UploadFile = File(...), 
    user_data: tuple = Depends(get_user_id)
):
    user_id, token = user_data
    
    if file.content_type != "application/pdf":
        raise HTTPException(status_code=400, detail="File must be a PDF")

    # Initialize client with the USER'S token for RLS enforcement
    user_client = create_client(SUPABASE_URL, SUPABASE_ANON_KEY)
    user_client.postgrest.auth(token) # For DB RLS
    # Note: For storage specifically, we pass the token in headers 
    # if using raw requests, but the SDK handles it if initialized correctly.
    
    file_path = f"{user_id}/{file.filename}"
    file_content = await file.read()

    try:
        # The storage RLS will check if the 'user_id' in the path 
        # matches the 'auth.uid()' in the token
        response = user_client.storage.from_("faculty-docs").upload(
            path=file_path,
            file=file_content,
            file_options={"content-type": "application/pdf"}
        )
        return {"message": "Success", "path": file_path}
    except Exception as e:
        # This will trigger if the RLS policy is violated
        raise HTTPException(status_code=403, detail=f"Storage error: {str(e)}")
```

### 3. How the "Higher Power" Accesses Files
Since you want admins to see data for multiple people, the "Admin" would simply request the file path stored in your `faculty` table.

1.  **Metadata Storage:** When a user uploads a file, you should save the file path (e.g., `user_id/resume.pdf`) into a column in your `faculty` table.
2.  **Accessing:** 
    *   A **Regular User** calls `storage.from('faculty-docs').download(path)`. RLS checks if the path starts with their ID.
    *   An **Admin** calls the same function. The "Admins can view all files" policy we wrote earlier will return `true`, allowing them to download it even if the ID doesn't match.

### Important Note on the "Service Role" Key
If you find yourself in a situation where the logic is too complex for SQL policies, you can use the `SUPABASE_SERVICE_ROLE_KEY` in your FastAPI app. This key **bypasses all RLS**. 

If you use that, you must be extremely careful to manually write `if user_role != 'admin':` checks in your Python code before calling any storage or database functions, otherwise you'll have a major data leak. **Sticking to RLS with the Anon Key + User Token is the "Gold Standard" for security.**
