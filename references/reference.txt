To build a secure FastAPI backend with Supabase, you'll want to leverage the **Supabase Python SDK** for database operations and its **Auth service** to verify the tokens sent from your frontend.

### 1. Project Setup
First, install the necessary libraries:
```bash
pip install fastapi uvicorn supabase python-multipart
```

### 2. The FastAPI Implementation
This example demonstrates how to verify a JWT, check for roles (e.g., an "admin" role for "higher power"), perform CRUD on a `faculty` table, and upload a PDF.

```python
import os
from fastapi import FastAPI, Depends, HTTPException, status, UploadFile, File
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
from supabase import create_client, Client

# --- CONFIGURATION ---
SUPABASE_URL = "https://your-project-id.supabase.co"
SUPABASE_KEY = "your-service-role-key"  # Use service_role for backend logic
supabase: Client = create_client(SUPABASE_URL, SUPABASE_KEY)

app = FastAPI()
security = HTTPBearer()

# --- AUTHENTICATION DEPENDENCY ---
async def get_current_user(credentials: HTTPAuthorizationCredentials = Depends(security)):
    """
    Verifies the JWT from the frontend and returns user data + role.
    """
    token = credentials.credentials
    try:
        # Validate the token with Supabase Auth
        user_response = supabase.auth.get_user(token)
        user = user_response.user
        
        # In Supabase, custom roles are often stored in app_metadata
        # You can set these via the Supabase Dashboard or an Edge Function
        role = user.app_metadata.get("role", "authenticated") 
        
        return {"id": user.id, "role": role}
    except Exception:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Invalid or expired token",
        )

# --- CRUD OPERATIONS ---

@app.get("/faculty")
async def get_faculty(user=Depends(get_current_user)):
    """
    Read: Users see their own data; 'admin' sees everything.
    """
    query = supabase.table("faculty").select("*")
    
    # Authorization logic
    if user["role"] != "admin":
        query = query.eq("user_id", user["id"])
        
    response = query.execute()
    return response.data

@app.post("/faculty")
async def create_faculty(name: str, department: str, user=Depends(get_current_user)):
    """
    Create: Add a new faculty record linked to the user.
    """
    data = {
        "name": name, 
        "department": department, 
        "user_id": user["id"]
    }
    response = supabase.table("faculty").insert(data).execute()
    return response.data

# --- FILE UPLOAD (PDF) ---

@app.post("/faculty/upload-cv")
async def upload_pdf(file: UploadFile = File(...), user=Depends(get_current_user)):
    """
    Uploads a PDF to Supabase Storage in a folder named after the user's ID.
    """
    if file.content_type != "application/pdf":
        raise HTTPException(status_code=400, detail="Only PDF files are allowed")

    file_path = f"{user['id']}/{file.filename}"
    file_content = await file.read()

    # Upload to a bucket named 'faculty-docs'
    try:
        response = supabase.storage.from_("faculty-docs").upload(
            path=file_path,
            file=file_content,
            file_options={"content-type": "application/pdf", "upsert": "true"}
        )
        return {"message": "Upload successful", "path": file_path}
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))
```

---

### 3. Supabase Setup Requirements

To make the above code work, ensure your Supabase project is configured as follows:

#### **Database Table**
Your `faculty` table should have a `user_id` column that links to `auth.users.id`.


#### **Storage Bucket**
1.  Go to **Storage** in your Supabase Dashboard.
2.  Create a new bucket named `faculty-docs`.
3.  Set the bucket to **Private** (recommended) so only authorized users can access the files.

#### **Roles & Security**
*   **Roles:** If you want to use roles like "admin," you can manually add them to a user's `app_metadata` using the Supabase SQL Editor:
    ```sql
    update auth.users 
    set raw_app_metadata_json = jsonb_set(raw_app_metadata_json, '{role}', '"admin"') 
    where email = 'admin@example.com';
    ```
*   **Service Role Key:** Since this is a backend, using the `service_role` key allows the FastAPI server to bypass RLS policies and handle the "Higher Power" logic (who sees what) directly in Python. 

### 4. How the Frontend interacts
The frontend simply needs to login via the Supabase client and pass the `access_token` in the header:
```javascript
// Example Frontend Header
const { data: { session } } = await supabase.auth.getSession();
fetch("http://localhost:8000/faculty", {
  headers: {
    'Authorization': `Bearer ${session.access_token}`
  }
});
```

Would you like to see how to implement the "Higher Power" logic using Supabase's native Row Level Security (RLS) instead of Python filters?
```


from fastapi import FastAPI, Depends, Header
from supabase import create_client, Client

SUPABASE_URL = "https://your-project-id.supabase.co"
SUPABASE_ANON_KEY = "your-anon-public-key"

app = FastAPI()

def get_supabase_client(authorization: str = Header(None)):
    """
    Creates a Supabase client that acts on behalf of the logged-in user.
    """
    if not authorization:
        return create_client(SUPABASE_URL, SUPABASE_ANON_KEY)
    
    # Extract the token from 'Bearer <token>'
    token = authorization.split(" ")[1]
    
    # Create client and set the auth token
    client = create_client(SUPABASE_URL, SUPABASE_ANON_KEY)
    client.postgrest.auth(token) 
    return client

@app.get("/faculty")
async def get_faculty(db: Client = Depends(get_supabase_client)):
    # With RLS enabled, this simple select(*) will automatically filter
    # data based on the user's ID or role!
    response = db.table("faculty").select("*").execute()
    return response.data
