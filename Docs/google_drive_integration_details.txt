# Google Drive Integration Details for Document Uploads (Part B)

This document outlines the non-code aspects and necessary modifications for integrating Google Drive to handle document uploads for all Part B API endpoints. The goal is to allow users to upload PDF documents via the frontend, have the backend store these documents in Google Drive, and then save the resulting Google Drive shareable link in the database.

## 1. Google Cloud Project Setup

This is a crucial one-time setup that needs to be performed in the Google Cloud Platform (GCP).

*   **Create a GCP Project:** If you don't have one, create a new project in the [Google Cloud Console](https://console.cloud.google.com/).
*   **Enable Google Drive API:** Navigate to "APIs & Services" -> "Library" and search for "Google Drive API". Enable it for your project.
*   **Create Service Account Credentials:**
    *   Go to "APIs & Services" -> "Credentials".
    *   Click "Create Credentials" and select "Service Account".
    *   Follow the steps to create a service account. Grant it appropriate roles (e.g., "Storage Object Creator" or a custom role with Drive access, or simply "Editor" for quick setup, but be mindful of permissions).
    *   After creation, click on the service account email, then go to the "Keys" tab.
    *   Click "Add Key" -> "Create new key" -> "JSON".
    *   A JSON file will be downloaded. This file contains your service account's private key and other credentials. **This file is highly sensitive and must be kept secure. Never commit it to version control or expose it publicly.**
    *   This JSON file will be used by the backend for authentication.

## 2. Google Drive Folder Setup

*   **Create a Dedicated Folder:** In your Google Drive, create a specific folder where all uploaded documents will be stored. This helps in organizing the files.
*   **Get Folder ID:** Once the folder is created, open it in Google Drive. The folder ID is part of the URL (e.g., `https://drive.google.com/drive/folders/YOUR_FOLDER_ID`). Copy this ID.

## 3. Environment Variables Configuration

To keep sensitive information out of the codebase, we will use environment variables.

*   **`.env` File:** Create or update your `.env` file in the project root with the following:
    ```dotenv
    GOOGLE_SERVICE_ACCOUNT_FILE=/path/to/your/service_account_key.json
    GOOGLE_DRIVE_UPLOAD_FOLDER_ID=YOUR_GOOGLE_DRIVE_FOLDER_ID
    ```
    *   Replace `/path/to/your/service_account_key.json` with the absolute path to the JSON key file you downloaded in step 1.
    *   Replace `YOUR_GOOGLE_DRIVE_FOLDER_ID` with the ID of the Google Drive folder you created in step 2.

## 4. Database Schema Modifications (Part B Endpoints)

For each Part B API endpoint that requires document uploads, the corresponding SQLAlchemy model and Pydantic schemas will need to be updated. Specifically, two new fields will be added:

*   **`department` (String):** To store the department associated with the entry.
*   **`document` (String):** To store the Google Drive `webViewLink` (the shareable URL) of the uploaded PDF document.

## 5. Backend Logic Overview

The FastAPI backend will perform the following sequence of operations:
1.  Receive the PDF file and other form data from the frontend via a `POST` request.
2.  Authenticate with Google Drive using the service account credentials loaded from the `.env` file.
3.  Upload the received PDF file to the specified Google Drive folder.
4.  Obtain the `webViewLink` (shareable URL) for the uploaded file.
5.  Store the `faculty_id`, `department`, `document` (Google Drive link), and other relevant data into the appropriate database table.
6.  Return a success response to the frontend.
